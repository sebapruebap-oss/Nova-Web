# Project Rules

## Objetivo

NovaWeb desarrolla proyectos web reales priorizando simplicidad, calidad y reutilización.

## Filosofía

Siempre resolver el problema más pequeño que permita avanzar.

Evitar complejidad innecesaria.

No incorporar nuevas tecnologías sin una necesidad concreta.

## Principios

- Mantener el proyecto simple.
- Mantener el código legible.
- Priorizar componentes reutilizables.
- Reducir duplicación.
- Reducir consumo de contexto.
- Documentar decisiones importantes.

## Antes de modificar

Preguntarse:

- ¿Cuál es el objetivo?
- ¿Qué archivos realmente necesito tocar?
- ¿Existe una Skill para esto?
- ¿Existe una Rule?

## Después de modificar

Siempre verificar:

- Mobile.
- Desktop.
- Consistencia.
- Código limpio.
