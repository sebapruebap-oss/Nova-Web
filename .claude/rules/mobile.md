# mobile.md — Experiencia responsive: mobile y desktop

## Principio general

La web está pensada principalmente para dispositivos móviles.
Mobile es la prioridad. Desktop se mejora sin romper mobile.

## Reglas de trabajo responsive

- Priorizar mobile en cada cambio.
- Revisar desktop después de cada cambio responsive.

## Validaciones — Mobile

Después de cualquier cambio, verificar en mobile:

- Header.
- Menú hamburguesa.
- Menú Productos.
- Navegación entre secciones.
- Carruseles táctiles.
- Vista parcial del siguiente producto cuando corresponda.
- Tarjetas.
- Modal.
- Botón cerrar.
- Variantes.
- Cantidad.
- Carrito.
- Checkout.
- WhatsApp.

## Validaciones — Desktop

Después de cualquier cambio, verificar en desktop:

- Header.
- Dropdown Productos.
- Hero.
- Centrado de secciones.
- Carruseles.
- Flechas.
- Tarjetas.
- Modal.
- Carrito.
- Secciones inferiores.
- Footer.

## Carruseles — comportamiento responsive

- En mobile, conservar la navegación táctil.
- No impedir que se vea parcialmente el siguiente producto si el diseño actual lo utiliza como señal visual.
- No cambiar anchos de tarjetas sin probar mobile y desktop.

## Modal — comportamiento responsive

El modal debe conservar su comportamiento responsive en todos los tamaños de pantalla.
