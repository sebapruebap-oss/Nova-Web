# Mobile Rules

## Objetivo

El diseño mobile es la prioridad principal de NovaWeb.

Toda nueva funcionalidad debe sentirse natural en un teléfono antes de optimizarse para desktop.

---

# Filosofía

No adaptar desktop a mobile.

Diseñar pensando en mobile desde el principio.

Desktop es una expansión del diseño, no el diseño principal.

---

# Prioridades

Cuando exista un conflicto, el orden de prioridad será:

1. Usabilidad
2. Legibilidad
3. Rapidez
4. Consistencia
5. Estética

Nunca sacrificar las primeras cuatro por una mejora únicamente visual.

---

# Espaciado

Cada sección debe respirar.

Evitar:

- bloques demasiado juntos
- márgenes inconsistentes
- pantallas visualmente pesadas

Si una pantalla genera sensación de saturación, reducir elementos antes que reducir espacios.

---

# Texto

Los usuarios no leen.

Escanean.

Por eso:

- títulos cortos
- párrafos breves
- botones claros
- información importante primero

Evitar bloques largos de texto.

---

# Botones

Todo botón debe poder tocarse cómodamente con un dedo.

Debe existir suficiente separación entre acciones distintas.

La acción principal debe destacar naturalmente.

---

# Tarjetas de producto

El usuario debe entender una tarjeta en menos de tres segundos.

Cada tarjeta debe responder rápidamente:

- Qué producto es.
- Cuánto cuesta.
- Cómo comprarlo.

Evitar agregar elementos que distraigan del producto.

---

# Imágenes

El producto siempre es el protagonista.

No recortar información importante.

Mantener proporciones naturales.

Usar fondos consistentes.

No modificar el producto cuando solo se trabaja el fondo.

---

# Responsive

Todo cambio debe verificarse en:

- teléfonos pequeños
- teléfonos grandes

No asumir que porque funciona en un dispositivo funcionará en todos.

---

# Performance

Evitar:

- imágenes innecesariamente grandes
- animaciones pesadas
- efectos que afecten la fluidez

La velocidad forma parte del diseño.

---

# Antes de terminar

Verificar:

□ No hay scroll horizontal.

□ No existen elementos cortados.

□ No existen botones difíciles de tocar.

□ No hay textos superpuestos.

□ El contenido mantiene una jerarquía visual clara.

□ La navegación es intuitiva.

---

# Regla de oro

Si una decisión mejora la estética pero empeora la experiencia del usuario, debe descartarse.
