# NovaWeb OS Constitution

## Misión

NovaWeb OS existe para capturar experiencia, convertirla en conocimiento reutilizable y usarla para construir software cada vez mejor, independientemente de la herramienta o modelo de IA utilizado.

## Principios

1. Resolver antes que impresionar.
2. La simplicidad gana.
3. Antes de automatizar, entender.
4. El conocimiento pertenece a NovaWeb.
5. Todo lo repetido se convierte en sistema.
6. La IA es un miembro del equipo, no un oráculo.
7. Cada proyecto mejora NovaWeb OS.
8. Construimos para el futuro.
9. Todo cambio importante debe poder explicarse.
10. La sesión no termina cuando el código funciona; termina cuando el sistema aprendió.

## Jerarquía

Constitución > Rules > Skills > Patterns > Workflows > Código

Si una Skill contradice una Rule, gana la Rule.
Si una Rule contradice la Constitución, gana la Constitución.

## Ciclo NovaWeb

1. Objetivo
2. Construcción
3. Validación
4. Aprendizaje
5. Captura del conocimiento

## Cierre de cada iteración

Al terminar una sesión importante, responder:

1. ¿Qué salió bien?
2. ¿Qué salió mal?
3. ¿Qué aprendimos?
4. ¿NovaWeb OS debería recordarlo?
5. ¿Qué parte de este trabajo debería volverse innecesaria la próxima vez?
